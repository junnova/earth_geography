# 需求文档：Earth Culture — 3D 手绘风格文化地球

## 1. 背景与目标

- **业务背景**：打造一个以地球为载体的文化展示平台，以梦幻手绘风格呈现全球各国的文化标志、著名景点和建筑。
- **要解决的问题**：以直观、有趣的方式让用户浏览和了解世界各国的标志性文化元素。
- **预期效果**：用户可以流畅地旋转、缩放一个手绘风格的 3D 地球，点击浮于地球表面的文化图标查看详情，也可以搜索国家快速定位。

## 2. 功能范围

### 2.1 包含功能

- [ ] 3D 地球渲染（手绘/梦幻风格底图）
- [ ] 地球流畅拖拽旋转 + 滚轮缩放
- [ ] Idle 状态下地球缓慢自转
- [ ] 2D 文化图标浮于地球表面（按国家，约 **195 个**点位）
- [ ] 点击图标弹出信息卡片（展示国家名、文化标志名称、图片、简介）
- [ ] 国家搜索功能（输入国家名，地球旋转定位到该国）
- [ ] 后台管理系统（手动添加/编辑/删除图标点位数据）
- [ ] 中文界面

### 2.2 不包含功能（当前版本）

- ❌ 移动端适配（后续迭代）
- ❌ 多语言支持
- ❌ 分类筛选
- ❌ 3D 模型图标
- ❌ 用户注册/登录体系

## 3. 用户场景

| 场景 | 用户操作 | 系统响应 |
|------|----------|----------|
| 自由浏览 | 打开页面 | 地球缓慢自转，展示手绘风格地球和各国文化图标 |
| 拖拽旋转 | 鼠标拖拽地球 | 地球跟随鼠标方向流畅旋转，停止拖拽后有惯性衰减 |
| 缩放查看 | 滚轮缩放 | 地球平滑缩放，图标大小保持可识别 |
| 查看文化详情 | 点击某国图标 | 弹出信息卡片，展示该国文化标志的名称、图片、简介 |
| 搜索国家 | 在搜索框输入"法国" | 地球平滑旋转至法国位置，高亮该国图标 |
| 关闭卡片 | 点击卡片外区域或关闭按钮 | 信息卡片关闭，恢复浏览状态 |
| 后台管理 | 管理员在后台添加一条数据 | 填写国家、经纬度、图标、名称、图片、简介后保存，前端实时可见 |

## 4. 验收标准

- [ ] 地球旋转帧率 ≥ 55fps（中等配置 PC，Chrome 浏览器）
- [ ] 拖拽旋转响应延迟 < 16ms（一帧以内）
- [ ] 拖拽停止后有自然惯性衰减效果
- [ ] 地球底图为手绘/梦幻风格，非写实卫星图
- [ ] 所有约 195 个国家图标可正常显示且不重叠
- [ ] 图标始终面向摄像机（Billboard 效果）
- [ ] 点击图标弹出卡片，信息完整展示
- [ ] 搜索输入国家名称后，地球在 1 秒内平滑旋转到目标位置
- [ ] 后台管理系统可 CRUD 所有图标数据
- [ ] Idle 3 秒后自动恢复缓慢自转

## 5. 技术要点

### 5.1 技术选型

| 层次 | 技术 | 理由 |
|------|------|------|
| **3D 渲染** | **Three.js** | 最灵活的 WebGL 库，社区成熟，手绘风格自定义度高 |
| **前端框架** | **Vue 3 + Vite** | 轻量快速，适合独立项目，搜索/卡片等 UI 组件开发高效 |
| **UI 组件** | **Element Plus** | 后台管理系统现成组件 |
| **后端** | **Node.js + Express** | 轻量 REST API，配合前端 JS 技术栈统一 |
| **数据库** | **SQLite**（开发）/ **MySQL**（生产可选） | 数据量小（~200 条），SQLite 零配置即可启动 |
| **地球底图** | 自定义手绘风格纹理贴图 | 使用水彩/插画风格的地球纹理图 |

### 5.2 关键技术点

- **流畅旋转**：使用 `OrbitControls` + `enableDamping`（惯性阻尼），`requestAnimationFrame` 驱动渲染循环
- **Billboard 图标**：使用 `THREE.Sprite` 或自定义面片始终朝向相机
- **经纬度转 3D 坐标**：球面坐标公式 `(lat, lon) → (x, y, z)`
  - `x = R * cos(lat) * cos(lon)`
  - `y = R * sin(lat)`
  - `z = R * cos(lat) * sin(lon)`
- **搜索定位动画**：使用 Tween/GSAP 驱动相机位置平滑动画
- **图标遮挡**：地球背面图标需降低透明度或隐藏，避免视觉混乱

### 5.3 潜在风险

| 风险 | 影响 | 应对 |
|------|------|------|
| 手绘风格底图获取 | 需要高质量等距柱状投影的手绘地图纹理 | 可使用开源素材或 AI 生成，备选方案：着色器后处理实现水彩效果 |
| 195 个图标同时渲染性能 | 低端设备可能卡顿 | 使用实例化渲染 / LOD，远距离时合并或隐藏 |
| 图标密集区域重叠（如欧洲） | 图标挤在一起不可识别 | 缩放后才展开密集区域，或聚合显示 |

### 5.4 性能要求

- 首屏加载 ≤ 3s（纹理贴图需压缩）
- 60fps 流畅交互（中等配置 PC）
- 纹理贴图总体积 ≤ 5MB

## 6. 数据结构

后台管理系统需要维护的数据字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | Integer | 自动 | 主键 |
| `country_name` | String | ✅ | 国家名称（中文），如"法国" |
| `country_code` | String | ✅ | ISO 3166-1 alpha-2 国家代码，如"FR" |
| `latitude` | Float | ✅ | 纬度，如 48.8566 |
| `longitude` | Float | ✅ | 经度，如 2.3522 |
| `landmark_name` | String | ✅ | 文化标志/景点名称，如"埃菲尔铁塔" |
| `description` | Text | ✅ | 简介（200 字以内） |
| `icon_url` | String | ✅ | 地球上显示的 2D 图标 URL |
| `image_url` | String | ❌ | 信息卡片中展示的大图 URL |
| `category` | String | ❌ | 类别：景点/建筑/文化符号（预留字段） |
| `created_at` | DateTime | 自动 | 创建时间 |
| `updated_at` | DateTime | 自动 | 更新时间 |
